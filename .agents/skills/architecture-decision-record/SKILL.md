# Architecture Decision Record

## Workflow

1. Read `AGENTS.md`, `agent/architecture/README.md`, and `agent/architecture/adrs/README.md`.
2. Gather the decision context from docs and code.
3. Compare realistic options and trade-offs.
4. Create or update an ADR using `agent/architecture/adrs/adr_template.md`.
5. Keep the ADR factual, concrete, and short enough to be maintained.
6. Link related product/architecture docs and implementation files where useful.

Do not invent decision history. If the rationale is not documented or verifiable, mark it as an explicit assumption.
