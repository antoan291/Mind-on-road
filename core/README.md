# Core Knowledge Layer

Тази папка е преносимото ядро за знания, източници, инженерни уроци и research бележки.

Целта е при следващ проект да може да се премести само `core` и оттам да се продължи с подредени източници, записани решения, грешки и research бележки.

## Структура

- `materials/books` — книги и PDF материали
- `materials/references` — допълнителни референтни файлове
- `research` — синтезирани бележки по теми
- `logs/version_journal.md` — версионен журнал на важните действия
- `logs/error_and_lessons_log.md` — грешки, последствия и постоянни правила
- `source_registry.md` — централен списък на книги, GitHub източници и важни линкове

## Работен принцип

1. Всяка по-голяма промяна се записва като версия.
2. Всяка грешка се записва с причина и правило как да не се повтори.
3. Важно външно знание се записва в source_registry.md и после се синтезира в research.
4. Код, архитектура и UI решения не трябва да разчитат само на чат памет; те трябва да оставят следа в docs и core.

## За този проект

Тази папка е съобразена с продукта за управление на автошколи и ще служи като reusable knowledge base за следващи SaaS проекти.
